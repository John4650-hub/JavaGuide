The following recommendations are all books I've read and I think are worth recommending.

These books, however, are more theoretical and can only help you to establish a standard of awareness for writing good code. If you want to develop better code and better software, you should look at good source code and learn good coding practices.

# The way the code is clean

**[Refactoring](https://book.douban.com/subject/30468597/)**

(https://oss.javaguide.cn/github/javaguide/books/202282814577.png)

It's a book! No need to talk.

Martin Fowler, the world's top national treasure, is the author of one of the most classic books in software development. The second edition has been released.

It's a book worth reading many times.

**[Clean Code](https://book.douban.com/subject/4199741)**

(https://oss.javaguide.cn/github/javaguide/books/202281748891.png)

Clean Code, a classic by Uncle Bob, is strongly recommended for all developers.

Mr. Bob has condensed his understanding of clean code in this book, which is a great gift to later generations.

**[Effective Java](https://book.douban.com/subject/30412517/)**

![https://oss.javaguide.cn/github/javaguide/books/82d510c9513883b325004802af6c0a.png]

The book "Effective Java" is a Java national treasure book, very classic. Java programmers must read it!

The book focuses on a number of very useful empirical rules in Java programming that cover the solutions to the problems most developers face on a daily basis. This book is very practical in helping you write clearer, stronger, and more efficient code. Each rule in this book appears in the form of a short, stand-alone article and is further illustrated by example code.

**[Code Complete](https://book.douban.com/subject/1477390/)**

(https://oss.javaguide.cn/github/javaguide/books/20210314173253221.png)

Actually, I don't really want to recommend this book. But due to its high bean petal rating, let's talk about it!

It's also a very classic book, and the second edition rewrites the first edition.

I have simply looked through the full text, and I feel that it is generally less useful for most programmers. If you want to actually improve the quality of your code, I think it's better than "The Art of Writing Readable Code."

However, it is important to look at good source code and learn good coding practices.

**[The Art of Writing Readable Code](https://book.douban.com/subject/10797189/)**

(https://oss.javaguide.cn/github/javaguide/books/202103145536443.png)

The meaning of this book, "The Art of Writing Readable Code," is very similar to "Clean Code," as you can see from their catalogs.

!\[https://oss.javaguide.cn/github/javaguide/books/watermark, type_ZmFuZ3poZW5naVpdGk, shadow_10, text_aHR0(M6Ly9ibG9nNzG4UbmV3FXzM0MzM3Mjcy, size_16,color_IF,t_70-20303030303033963.png)

It seems to me that if you read "Clean Code," you don't need to read it anymore. Of course, if you have time and energy, you can go over it quickly.

In addition, I would like to recommend a repository called **[write-readable-code](https://github.com/biezhi/write-readable-code)**. The authors of this repository shared, free of charge, a series of videos based on the book "The Art of Writing Readable Code." This series of videos is based on the Java language to teach you how to optimize your code.

It'll be better to learn in practice! Let's get to work!
