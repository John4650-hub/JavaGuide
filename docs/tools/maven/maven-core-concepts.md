# Maven Introduction

[Maven](https://github.com/apache/maven) The official document describes Maven as:

> Apache Maven is a software project management and control tool. Based on the concept of a project object (POM), Maven can manage a project's build, reporting, and documentation from a central piece of information.
>
The essence of Apache Maven is a software project management and understanding tool. Based on the concept of the Project Object Model (Project Object Model, POM), Maven can build, report, and document from a central information management project.

**What is POM?** Every Maven project has a `pom.xml` file in the root directory containing detailed information on the life cycle of the project's construction. Through the `pom.xml` file, we can define the project coordinates, project dependencies, project information, plugin information, etc.

For developers, the main roles of Maven are three:

1. **Project construction**: providing standard, cross-platform automated project construction.
2. **Dependency management**: easy and quick management of the resources (jar packages) on which the project depends to avoid the problem of conflicting versions between resources.
3. **Harmonized development structure**: provides a standard, harmonized project structure.

The basic use of Maven is not presented here, and it is suggested to look at the five minutes of Maven's course on the official network: [Maven in 5 Minutes](https://maven.apache.org/guides/getting-started/maven-in-five-minutes.html).

# Maven Coordinate

The third-party repositories and plugins relied on in the project may be collectively referred to as the building blocks. Each component can use the single marking of the Maven coordinates, which include:

- **groupId** (must): defines the organization or company to which the current Maven project belongs. GroupId is generally divided into multiple sections, usually the first is the domain and the second is the name of the company. The domain is divided into the following fields: org, com, cn, etc., of which org is a non-profit organization, com is a business organization, and cn means China. In the case of the Apache open-source community Tomcat project, the project's groupId is org.apache, its domain is org (because Tomcat is a not-for-profit project) and the name of the company is Apache, artifactId is tomcat.
- **artifactId** (must): The name of the current Maven project, the only identifier for the project, corresponds to the name of the root directory of the project.
- **version** (must): The current version of the Maven project that has been defined.
- **packaging** (optional): Defines the packaging method for the Maven project (e.g., jar, war...), defaulted to jar.
- **classifier** (optional): often used to distinguish between components with different elements constructed from the same POM, which may be an arbitrary string attached to the version number.

As long as you provide the right coordinates, you can find the corresponding components from the Maven warehouse for us.

For example (EasyExcel, which introduced by Alibaba):

```xml
<dependency>
  <groupId>com.alibaba</groupId>
  <artifactId>easyexcel</artifactId>
  <version>3.1.1</version>
</dependency>
```

You can find almost all available components on this website at <https://mvnrepository.com/>, and if your project is using Maven as a build tool, then this site will be in constant contact.

# Maven Dependency

If the building component (e.g., the Jar file) generated by using Maven is cited by other projects, then the component is dependent on other projects.

# Dependency Configuration #

**Examples of configuration information**:

```xml
<dependencies>
  <dependency>
    <groupId>org.example</groupId>
    <artifactId>example-artifact</artifactId>
    <version>1.0.0</version>
    <exclusions>
      <exclusion>
        <groupId>org.example.excluded</groupId>
        <artifactId>excluded-artifact</artifactId>
      </exclusion>
    </exclusions>
  </dependency>
