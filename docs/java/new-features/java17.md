# JEP 356: Enhanced Pseudo-Random Generator

Before JDK 17, we could generate random numbers using `Random`, `ThreadLocalRandom`, and `SplittableRandom`. However, each of these three categories is flawed and lacks common pseudo-random algorithm support.

Java 17 added new interface types and implementations to the pseudo-random generator (pseudorandom number generator, PRNG, also known as the deterministic random number generator), making it easier for developers to use various PRNG algorithms in applications.

> [PRNG](https://ctf-wiki.org/crypto/streamcipher/prng/intro/) is used to generate numerical sequences close to absolute random series. Generally, PRNG relies on an initial value, also known as a seed, to generate the corresponding pseudo-random sequence. The random numbers generated by PRNG are fully established as long as the seeds are established, so the random series generated by PRNG is not really random.

Example used:

```Java
RandomGeneratorFactory<RandomGenerator> l128X256MixRandom = RandomGeneratorFactory.of("L128X256MixRandom");
// Use timestamp as a random seed
RandomGenerator randomGenerator = l128X256MixRandom.create(System.currentTimeMillis());
// Generate Random Number
int randomNumber = randomGenerator.nextInt(10);
```

# JEP 398: Deprecate Applet API for Removal

The Applet API for the Java applet that runs in the web browser has been phased out years ago and is no longer justified.

The Applet API was discarded in Java 9 ([JEP 289](https://openjdk.java.net/jeps/289)), but not for deletion.

# JEP 406: Switch Type Matching (Preview)

Like `instanceof`, `switch` has been enhanced by the addition of a type matching automatic conversion feature.

Example of `instanceof` code:

```Java
// Old code
if (o instanceof String) {
    String s = (String) o;
    // ...use s...
}

// New code
if (o instanceof String s) {
    // ...use s...
}
```
