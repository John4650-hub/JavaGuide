# Foundation

## Web Frame

- [Spring Boot](https://github.com/spring-projects/spring-boot "spring-boot"): Spring Boot can easily create an independent production-grade Spring application with a built-in web server that allows you to run items like a normal Java program. In addition, most Spring Boot projects require only a small amount of configuration, which is different from Spring's reconfiguration.
- [SOFABoot](https://github.com/sofastack/sofa-boot): SOFABoot is based on Spring Boot but adds to it the capacity of Readiness check, class isolation, log space segregation, etc. Also available are the SOFARPC (RPC) framework, SOFABolt (based on Netty's remote communication framework), SOFARegistry (Registration Centre)... For further information, see [SOFAStack](https://github.com/sofastack).
- [Solon](https://gitee.com/opensolon/solon): The Java enterprise-level application development framework for national production for the whole scene.
- [Javalin](https://github.com/tipsy/javalin): A lightweight web framework supported by Java and Kotlin, used by Microsoft, Red Hat, Uber, etc.
- [Play Platform](https://github.com/playframework/playframework): High-speed web framework for Java and Scala.
- [Blade](https://github.com/lets-blade/blade): A simple, efficient web framework based on Java 8 + Netty 4.

## Micro-services / Clouds

- [Armeria](https://github.com/line/armeria): A framework for micro-services appropriate to any situation. You can build any type of micro-services using technology you like, including [gRPC](https://grpc.io/), [Thrift](https://thrift.apache.org/), [Kotlin](https://kotlinlang/), [Retrofit](https://square.github.io/retrofit/), [Reactive Streams](https://www.reactive-streams.org/), [Spring Boot](https://spring.io/projects/spring-boot) and [Dropwizard](https://www.dropwizard.io/).
- [Quarkus](https://github.com/quarkusio/quarkus): A framework for the preparation of clouds and containers for Java applications.
- [Helidon](https://github.com/helidon-io/helidon): A group of Java libraries used to develop microservices that support two programming models, Helidon MP and Helidon SE.

## API Document

- [Swagger](https://swagger.io/): API document tool in a more mainstream RESTful style provides a set of tools and specifications that enable developers to create and maintain more easily readable, user-friendly, and interactive API documents.
- [Knife4j](https://doc.xiaominfo.com/): A collection of Swagger2 and OpenAPI3 enhanced solutions.

# Bean Map

- [MapStruct](https://github.com/mapstruct/mapstruct) (recommended): A Java Note Processor that meets the JSR269 standard, which is used to generate a safe and high-performance type of map for Java Bean. It is based on the generation of the get/set code at the compilation stage, which is achieved without reflection and without additional performance loss.
- [JMapper](https://github.com/jmapper-framework/jmapper-core): A high-performance and easy-to-use Bean mapping framework.

## Other

- [Guice](https://github.com/google/guice): A lightweight type-dependent injection framework from Google's open source, equivalent to a very simple lightweight Spring Boot. In some cases, it is very practical, for example, our projects need only use dependency injections, without functional features such as AOP.
- [Spring Batch](https://github.com/spring-projects/spring-batch): Spring Batch is a lightweight but fully functional batch-processing framework that is used mainly for batch scenes such as reading a large number of records from databases, files, or queues. However, it needs to be noted that Spring Batch is not a scheduling framework. There are many excellent business movement frameworks in both commercial and open-source areas such as Quartz, XXL-JOB, and Elastic-Job. It is intended to work with, and not to replace, the movement procedures.
