## Scene One: The Order System

We buy things at the mall, we pay with two-dimensional codes, the order numbers generated by the tabs, the coupons used, the coupons for joint goods, and the single numbers that are often used in online shopping. The following is an analysis of the practical realization of the ID services for the different scenarios in the order system.

### 1. One-Yard Payment

The common one-yard payment is a two-dimensional code that can be used to scan the money with a payment treasure or a microlink.

The essence of a 2D code is a string. The essence of the polymer code is a link address. Users use the payment code to sweep the payment code directly, without fear of using the payment code to sweep the payment code or the payment code to sweep the payment code, which greatly reduces the time spent on the payment code.

The principle of realization is that when a customer uses an APP scan, the backend of the website will judge the client's scan environment. (We believe, pay for treasures, QQ Q wallets, Kyoto payments, clouds flashing, etc.)

The rationale for judging the sweep environment is based on the HTTP header that opens the link browser. When any browser opens the HTTP link, the requested header will have User-Agent (UA).

UA is a special string header, and the server in turn recognizes a lot of information such as the operating system and version used by the client, CPU type, browser and version, browser rendering engine, browser language, browser plugin, etc.

The name of the payment product varies from one channel to another, and it is important to look closely at the API description of the payment product.

1. Micro-credit payments: JSAPI payments

### 2. Order Number

The order number exists as the only identification number for an order in the actual course of business, generally achieving the following business scenarios:

1. User order problems requiring assistance in finding a customer's uniform;
1. Operation of orders, such as under-line collections and cancellation of orders;
1. Order processing and follow-up within the system of placing orders, changing orders, making orders, refunds, etc.

In many cases, the order information is searched with the order ID as the only identifier because the order number is the only one that determines the order creation rules.
